最近在面试过程中，有问道java值传递的问题，一开始脑子里想的就是不就是传值吗？一个int类型数据作为参数传入函数，那么函数中就是拿的它的值，对它的值进行修改并不会影响它本身。对于对象也是传递的值...一时间突然不知道这个值指的是什么，于是我准备再学习一下这个知识点。

<!-- more -->

### 基础数据

看这样一个列子：

```java
public class BasePassValue {

    public void basePassValue(int x) {
        x = 10;
        System.out.println("basePassValue x = " + x);
    }

    public static void main(String[] args) {
        BasePassValue pv = new BasePassValue();
        int x = 1;
        System.out.println("start x = " + x);
        pv.basePassValue(x);
        System.out.println("end x = " + x);
    }
}
```

输出结果：

```
start x = 1
basePassValue x = 10
end x = 1
```

可以看出，对于基础数据，传递的就是基础数据值本身。

### 对象

继续看代码：

```java
public class ObjectPassValue {

    private float x;

    public void objectPassValue(ObjectPassValue pv) {
        pv.x = 3f;
        System.out.println("objectPassValue pv = " + pv + ", x = " + pv.x);
        pv = null;
        System.out.println("objectPassValue pv = " + pv);
    }

    public static void main(String[] args) {
        ObjectPassValue pv = new ObjectPassValue();
        pv.x = 2f;
        System.out.println("start pv = " + pv + ", x = " + pv.x);
        pv.objectPassValue(pv);
        System.out.println("end pv = " + pv + ", x = " + pv.x);
    }
}
```

输出结果：

```
start pv = com.zmt.java8.java.passbyvalue.ObjectPassValue@a57993, x = 2.0
objectPassValue pv = com.zmt.java8.java.passbyvalue.ObjectPassValue@a57993, x = 3.0
objectPassValue pv = null
end pv = com.zmt.java8.java.passbyvalue.ObjectPassValue@a57993, x = 3.0
```

可以看出，传入函数的对象在函数调用前后的对象引用没有变化，都是a57993，所以传入函数中的不是对象本身，而是指向该对象的引用副本，或者说传入了对象的引用地址值，函数里面的对象变量指向了该地址值，所以对象变量的属性值x对传入函数的对象也起作用，因为二者指向的内存地址一样。然后将函数里面的对象设置为null后，就断掉了对象变量的引用地址指向，并且也不影响传入函数的对象。

### 总结

简单来说，对于基础数据，值传递传递的是基础数据值，函数内部的操作不影响原本的参数；

对于对象，值传递的是引用地址值，函数内部的变量会执行该引用地址，对象本身的操作不会影响原本的对象，对象属性的修改会影响原本的对象。
